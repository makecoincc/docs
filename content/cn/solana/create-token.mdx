Solana 上的代币被称为 SPL (Solana Program Library) 代币。Solana 生态系统有两个主要的 Token Program。

- [Token Program (原始版本)](https://github.com/solana-program/token) 基本代币功能（铸造、转移等）不可变且广泛使用
- [Token Extension Program (Token 2022)](https://github.com/solana-program/token-2022) 包含所有原始 Token Program 功能 通过“扩展”添加功能

创建代币就是使用solana的客户端程序，调用这两个中的一个。

从创建代币mint帐户开始，一般还需要铸币，最终在用户钱包中显示。所以创建币有几个相应的步骤

### Token Program (原始版本)
<Callout>提供代币的Decimals和Supply，创建代币</Callout>
- 生成铸币账户和代币账户密钥对。
- 创建代币 mint 帐户，并初始化。
- 创建token account并初始化。
- 创建关联token account。
- 铸造代币到关联token account。
- 撤销铸币权限（可选）通常在初始供应创建后撤销，以创建固定供应的代币。
- 撤销冻结权限（可选）通常会被撤销，以保证用户的代币不能被冻结。


### Token Extension Program (Token 2022)
<Callout>提供代币的Decimals、Supply、Name、Symbol和Uri，创建代币</Callout>
- 生成铸币账户和代币账户密钥对。
- 组装metadata。（Token 2022才有）
- 计算创建帐户所需要的空间和费用。（Token 2022才有）
- 创建代币 mint 帐户，并初始化。
- 初始化meta pointer extension。（Token 2022才有）
- 初始化 meta extension。（Token 2022才有）
- 创建token account并初始化。
- 创建关联token account。
- 铸造代币到关联token account。
- 撤销铸币权限（可选）通常在初始供应创建后撤销，以创建固定供应的代币。
- 撤销冻结权限（可选）通常会被撤销，以保证用户的代币不能被冻结。

### 说明

在创建代币过程中创建的mint帐户和token account帐户，都不需要保留私钥，因为初始化mint时，分配了mint和freeze权限给钱包帐户，初始化token account时，将owner设为了钱包帐户。丢弃它们反而是安全的。


### 命令行工具
如果你在电脑上安装了[Solana CLI](https://solana.com/zh/docs/intro/installation)，以上创建代币也可以使用[spl-token](https://solana.com/zh/docs/tokens#%E5%88%9B%E5%BB%BA%E6%96%B0%E4%BB%A3%E5%B8%81)命令完成，包括token 2022
